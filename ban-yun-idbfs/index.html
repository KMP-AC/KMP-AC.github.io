搬运-IDBFS

2020-05-06 BFS

D-BFS就是以深搜执行代替广搜，因为他们都有一个性质:

  • 以当前节点进行操作

那么广搜是采用扩展操作，深搜就是迭代操作。

给个例子：

这个是网图，你可以看到，这个图广搜工作量比较多，深搜工作量比较少。

那么，就有一些毒瘤出题人会卡你的广搜，深搜直接搜到最短路.

那么此处，给出一个深搜版广搜的伪代码：

queue<int>queue;
int ans=0,t;
void Bfs(int x,int deep)
{
  if(x==t) ans=deep,return;
  if(deep==ans||queue.empty()) exit(0);
  for(扩展）
  queue.pop();
  BFS(队头)
}

这个不是很鸡肋吗，我广搜吊打他。

的确很鸡肋，但我还有一个更好的。

比如说最短路，BFS只能解决无权，如果可以解决有权，整么办？

这时候，如果你想反悔，回到一个点，那么你可以整么办？回溯！这是你的脑子里会不会想到这个?

没错，我们可以这样想:写一个数组来记录状态！这样我们就可以回去某一步！

但是这还不够，我们需要一个严格正确的判断来进行优化！于是启发式搜索又来了！

我们可以想贪心(骗分嘛）策略,选一条边权最小的边！

我们可以在建图时优化，还可以使用O(NlogM)O(NlogM)O(NlogM)的快排或者O(M)O(M)O(M)的基排。

然后有会有一种数据卡爆你！就是垫底，然后一个负权!

所以，这个也不行!

ID-BFS

没错，迭代加深！如果每次都跑不过一个K,那扩展干什么?

放弃正确性!考虑一个贪心最简单最错误却最让人想不到！

  • 如果某个点F的最短路已经是最短的，那么设置上限为最短+次短!

那么，这当然是一个极为不错的主意，其实会被卡爆...

但是这至少比SPFASPFASPFA要快！

如果要更快，可以选择BFS提前O(N)O(N)O(N)遍历一下，可以优化回溯操作。
